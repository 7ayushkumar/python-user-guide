<h3 id="What-is-Plotly?">
        What is Plotly?
        <a class="anchor-link" href="#What-is-Plotly?">
         ¶
        </a>
</h3>
</div>
</div>
</div>
<div class="cell border-box-sizing text_cell rendered">

<div class="inner_cell">
<div class="text_cell_render border-box-sizing rendered_html">
<p>
        Answer from Plotly CEO &amp; Co-Founder Jack Parmer:
       </p>
<blockquote>
<p>
         Plotly is like graphing crack. It standardizes the graphing interface across scientific computing languages (Python, R, MATLAB, etc) while giving rich interactivity and web shareability that has not been possible before with matplotlib, ggplot, MATLAB, etc.
        </p>
<p>
         On the Plotly website, you can style your graphs with a GUI, so you don't have to spend hours writing code that simply changes the legend opacity.
        </p>
<p>
         Plotly does this all while backing up your graphs on the cloud, so that years later, you can find data that may have otherwise been on a harddrive in a landfill. If you make your data public, other people can also find your graphs and  data. The best practice that we have today for saving and sharing research data is to entomb it as a thesis in the engineering library basement. All that is changing.
        </p>
<p>
         Like
         <a href="http://d3js.org/" target="_blank">
          d3.js
         </a>
         ? Like interactive,
         <a href="https://plot.ly/~bchartoff/344/" target="_blank">
          NYT graphics
         </a>
         ? So do we. Now, with the Plotly APIs, you can make them yourself without being an expert web programmer. If you are an expert web programmer, now you have scientific languages and tools like  R, Python, Pandas, and MATLAB instead of javascript to wrangle your data and create beautiful data vis. Science meets the world-wide-web. Engineering meets design. Let's do this.
        </p>
</blockquote>
</div>

<h3 id="Why-a-User-Guide?">

        <a class="anchor-link" href="#Why-a-User-Guide?">

<div class="text_cell_render border-box-sizing rendered_html">
<p>
        The User Guide project has the goal of giving Plotly's Python/IPython users extensive documentation on all of Plotly's features.
       </p>
<p>
        Whereas Plotly's web
        <a href="https://plot.ly/python" target="_blank">
         documentation
        </a>
        is meant to serve Plotly beginners
just looking to make a simple plot and experienced users trying out a new plot type, 
the User Guide intends to turn beginner and intermediate Plotly users into experts.
       </p>
<p>
        It is assumed that readers of the User Guide are
        <em>
         comfortable
        </em>
        with Python terminology (e.g. know what Python objects, lists, dictionaries and functions are). But, there is absolutely no need to be a Python guru to follow the User Guide's content.
       </p>
</div>
</div>
</div>

<p>
        Although not required, we recommend setting up a
        <em>
         credentials
        </em>
        file on your machine:
       </p>
<p>
        In Python or IPython,
       </p>
<pre><code>&gt;&gt;&gt; import plotly.tools as tls
&gt;&gt;&gt; tls.set_credentials_file(
        username="your_username", 
        api_key="your_api_key")
</code></pre>
<p>
        Where the
        <code>
         username
        </code>
        and
        <code>
         api_key
        </code>
        keyword arguments are filled in with your own as found in
        <a href="#Step-5">
         step 5
        </a>
        .
       </p>
<p>
        This creates a
        <code>
         .credentials
        </code>
        file in a folder called
        <code>
         $HOME/.plotly/
        </code>
        (where
        <code>
         $HOME
        </code>
        is your home directory) storing your username and API key locally in JSON file.
       </p>
<p>
        You can print your credentials to the stdout, inside Python or IPython, with:
       </p>
<pre><code>&gt;&gt;&gt; tls.show_credentials_file()
</code></pre>
<p>
        and
       </p>
<pre><code>&gt;&gt;&gt; my_creds = tls.get_credentials_file()
</code></pre>
<p>
        writes your Plotly credentials into a Python dictionary and assigns it to a variable.
       </p>

        </em>

       </p>
<p>
        While being imported, the
        <code>
         plotly.plotly
        </code>
        module looks for your credentials file on your machine and automatically uses it to sign in to Plotly's servers.
       </p>

        Using credentials files allows users to share code without having to type in (let alone remember) their own username and API key every time they want to generate a new Plotly plot or modify an existing Plotly plot.
       </p>
<p>
        That said, if more convenient, users can still manually sign in to Plotly by typing:
       </p>
<pre><code>&gt;&gt;&gt; py.sign_in('your_username','your_api_key')
</code></pre>
